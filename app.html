命令的使用

直接执行 7z 可以查看版本与帮助

解压文件

7z x C:\1.zip-oC:\test # #解压C盘下的 1.zip 到C盘下的test目录里（注意：o 后面没有空格）

7z x 1.zip-otest # 解压当前目录下的 1.zip 到当前目录下的 test 目录里

添加文件

7z a 1.7z # 添加当前目录下所有的文件到 1.7z 的压缩包（没有则创建）

7z a 1.7z test/ # 添加当前目录下 test 目录里的所有文件到 1.7z 压缩包里

7z a T:\Downloads\dokuwiki.zip W:\dokuwiki\data\pages # 将指定路径的目录压缩为指定路径的名为 dokuwiki.zip 的压缩包

删除文件

7z d 1.zip*.bak-r # 删除 1.zip 压缩包中的所有扩展名为 bak 的文件

选项

  • a：将文件添加到压缩档案中
  • b：测试压缩或解压算法执行时的CPU占用
  • d：从压缩档案中删除文件
  • e：将压缩档案中的所有文件解压到指定路径，所有文件将输出到同一个目录中
  • h：计算文件的哈希值
  • i：显示有关支持格式的信息
  • l：列出压缩档案的内容
  • rn：重命名压缩档案中的文件
  • t：测试压缩档案的完整性
  • u：更新要进入压缩档案中的文件
  • x：将压缩档案中的所有文件解压到指定路径，并包含所有文件的完整路径

    -m n # 最大下载连接数
    -d T:/Downloads/ # 指定目录
    -all-proxy="http://127.0.0.1:10809" # 指定代理
    --http-proxy=url 指定http代理
    --max-download-limit=1024K # 限速

# PowerShell
function hb {
    if ($args.Length -eq 0) {
        Write-Error "No file path specified."
        return
    }
    $FilePath = $args[0]
    if (Test-Path $FilePath) {
        $Content = Get-Content $FilePath -Raw
    } else {
        Write-Error "File path does not exist."
        return
    }
    $uri = "http://bin.christitus.com/documents"
    try {
        $response = Invoke-RestMethod -Uri $uri -Method Post -Body $Content -ErrorAction Stop
        $hasteKey = $response.key
        $url = "http://bin.christitus.com/$hasteKey"
        Write-Output $url
    } catch {
        Write-Error "Failed to upload the document. Error: $_"
    }
}

# Linux Bash
function hb {
    if [ $# -eq 0 ]; then
        echo "No file path specified."
        return
    elif [ ! -f "$1" ]; then
        echo "File path does not exist."
        return
    fi
    uri="http://bin.christitus.com/documents"
    response=$(curl -s -X POST -d "$(cat "$1")" "$uri")
    if [ $? -eq 0 ]; then
        hasteKey=$(echo $response | jq -r '.key')
        echo "http://bin.christitus.com/$hasteKey"
    else
        echo "Failed to upload the document."
    fi
}

    VirtualBox 中的网络连接说明

    桥接网卡：虚拟机成为局域网内与宿主机同等的存在，它们在同网段中可以互相访问
    网络地址转换（NAT）：该虚拟机能访问主机和网络，但宿主机不能访问该虚拟机
    NAT 网络 ：在 NAT 的基础上，虚拟机之间搭建了局域网，能够相互访问
    内部网络 ：虚拟机不能连外网，只能被属于同一主机 Host 中的其他虚拟机所访问
    仅主机（host-only）网络：虚拟机不能连外网，但宿主机和虚拟机可以访问它
    未指定：虚拟机中有网卡，但是没有插网线

    上传之前可以做如下配置：

    修改账户密码：默认有两个用户，admin 和 user，先访问 php 密码生成，然后替换 tinyfilemanager.php 中如下代码部分： 
    // Users: array('Username' => 'Password', ...)
$auth_users = array(
    'admin' => '替换这里', //admin（这是默认密码）
    'user' => '替换这里', //12345（这是默认密码）
);
    如果不想使用账户密码可以修改此部分：$use_auth = true;（改为 false 为禁用

    设定根目录：默认会访问服务器的根目录，我这里指定了 disk 目录，只想允许访问这个目录，那么可以修改如下部分：
    // Root path for file manager
// use absolute path of directory...
$root_path = $_SERVER['DOCUMENT_ROOT'].'/disk';（即 加入 .'/disk' ）
    隐藏文件： 
    
$exclude_items = array(
    'abc',（隐藏 abc 的文件夹）
    'a.mp4',（隐藏 a.mp4 文件）
    '*.php' （隐藏 php 扩展名的文件）
);
    ❌ 下载 binaries.tar.gz 并解压，其中 vlmcsd-Windows-x64.exe 是 KMS-Server，从 binaries/windows/intel 目录里摘出来，激活只要这一个文件，获得IP地址后，在要激活的机器上用命令激活

    vlmcs-Windows-x86.exe 用来测试 KMS-Serve 是否可用，后面参数带上IP地址即可（这步可以省略，正常使用第一个文件足够）：vlmcs-Windows-x86.exe 123.123.123.123，显示类似结果有 successful 表示KMS服务器正常运行

    注册为 Windows 服务

    将 binaries/Windows/intel 目录重命名为 vlmcsd 并放到C盘，在该目录中打开终端执行：./vlmcsd-Windows-x64.exe -s -l C:\vlmcsd\KMS.log # -s 为安装为服务，-S 是删除服务，-l 指定输出日志文件路径

    运行 - Services.msc 打开服务管理，找到 Key Management Server 服务器，启动服务并确保启动类型为自动，再去防火墙开放 1688/tcp 端口，最后执行 ./vlmcs-Windows-x64.exe 显示 Successful，表成功
